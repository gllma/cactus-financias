Toda mensagem voltada ao usuário DEVE usar `__('modulo.chave')` ou `trans()`.

- NUNCA string literal em responses, exceções ou emails
- Organizar por domínio: auth.php, trip.php, route.php, address.php, user.php, admin.php, notification.php, common.php
- Variáveis dinâmicas via `:placeholder`, NUNCA concatenação
- Chave DEVE existir nos 3 idiomas: pt_BR (default), es (fallback), en
- Tom humanizado e empático — o usuário é um cidadão PNE, não um dev
- Evitar mensagens genéricas ("Recurso criado com sucesso!"). Contextualizar sempre.

## Placeholders

NUNCA concatenar strings com `__()`. SEMPRE usar `:placeholder`:

```php
// ERRADO — quebra em idiomas com ordem de palavras diferente
__('modulo.mensagem') . " ({$campo})"

// CERTO — funciona em qualquer idioma
__('modulo.mensagem', ['campo' => $campo])
```

No arquivo de tradução:
```php
'mensagem' => 'Não é possível realizar a ação (:campo).',
```
